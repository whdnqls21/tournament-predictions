> 2026 FIFA 북중미 월드컵 친목 예측 내기 웹앱. 친구 4명(윤·준·경·빈)이 각자 폰에서
> 라운드별 승팀을 예측하고, 관리자가 결과를 입력하면 점수·등수·선물 정산이 자동 계산된다.

## 2. 예측 / 진행 규칙

- **예측 단위**: 라운드마다 대진이 확정된 뒤, 각 경기에서 **이길 팀(=다음 라운드 진출 팀) 하나**를 탭으로 선택. 스코어 예측 없음.
- **정답 판정**: **진출 여부**로 판정. 연장·승부차기로 올라가든 상관없이 다음 라운드로 간 팀 = 정답. 3·4위전은 이긴 팀 = 정답(판정 방식 동일).
- **라운드 진행 순서**: 32강 결과 입력 → 16강 예측 오픈 → 16강 결과 입력 → 8강 예측 오픈 → … 순차. **앞 라운드 결과가 입력돼야 다음 라운드 예측이 열린다.**
- **마감 조건**: 4명 **전원 확정** 시 자동 마감. 백업으로 **관리자 강제 마감** 가능(잠수 대비). 마감 시점에 확정 안 한 사람은 그 라운드 **0점**.
- **수정**: 확정을 눌러도 **라운드 마감 전**이면 "확정 취소" 후 다시 수정 가능. 마감되는 순간 전부 잠금.
- **예측 공개 범위**:
  - 마감 **전**: 남의 예측 내용은 가림. "누가 확정했는지" 여부만 표시.
  - 마감 **후**: 그 라운드 4명 예측 전부 공개.

## 3. 배점 / 채점 / 동점 / 선물

### 배점 (경기당 점수, 총 74점 만점)

| 라운드 | 경기 수 | 경기당 | 라운드 만점 |
|---|---|---|---|
| 32강 | 16 | 1점 | 16점 |
| 16강 | 8 | 2점 | 16점 |
| 8강 | 4 | 4점 | 16점 |
| 4강 | 2 | 6점 | 12점 |
| 결승 | 1 | 10점 | 10점 |
| 3·4위전 | 1 | 4점 | 4점 |
| **합계** | 32 | | **74점** |

> 32강(round of 32) ~ 결승. 4강 = 준결승(semifinal) 2경기. 결승·3·4위전은 준결승 결과로 자동 생성.

### 채점

- 각 경기: 참가자가 찍은 팀이 실제 진출(=정답)이면 해당 라운드 경기당 점수 획득.
- **입력된 라운드까지만** 점수 누적. 결과가 들어올수록 점수가 쌓인다.
- 미확정/미제출자는 해당 라운드 0점.

### 동점 처리 (순서대로)

1. 결승(우승팀)을 맞힌 사람 우선
2. 그래도 같으면 결승 진출 2팀 적중 수가 많은 사람
3. 그래도 같으면 공동 등수

### 선물 정산

- **자기보다 위 등수인 사람 각각에게 롤 "수수께끼 스킨" 1개씩** 선물.
  - 2등 → 1등에게 (1개)
  - 3등 → 1·2등에게 (2개)
  - 4등 → 1·2·3등에게 (3개)
  - 1등 → 받기만 (3개 수령)
- 공동 등수가 생기면 "자기보다 **엄격히 위**(higher) 등수"에게만 선물(동순위끼리는 주고받지 않음).
- 순위 화면에 "누가 누구에게 몇 개" 정산표를 자동 표시.

## 4. 인증 / 권한

- **참가자**: 이름(윤·준·경·빈) 선택 + **개인 4자리 PIN**. 본인 예측만 보기·수정.
- **PIN 발급**: 관리자가 셋업 때 4명 PIN을 지정해 각자에게 전달. **분실 시 관리자가 재설정**.
- **관리자**: 참가자와 **별개의 관리자 전용 PIN**으로 진입. (빈이 운영 예정이나, 신원은 분리)
- 관리자 기능: 32강 브라켓 셋업, 라운드 결과 입력, 강제 마감, 참가자 PIN 지정/재설정, 결과 되돌리기.

> 보안 수준: 4명 친목용. PIN은 강한 인증이 아니다. 단, **결과 입력·되돌리기 등 관리자 행위는 관리자 PIN 뒤에서만** 가능해야 함. 서버 측(API route 또는 Supabase RLS/secret)에서 관리자 PIN을 검증해 클라이언트 우회를 막을 것.

## 5. 대진(브라켓) / 결과 입력

- **셋업**: 관리자가 32강 16경기 대진을 **브라켓 자리에 배치**(누가 어느 위치인지). 토너먼트 경로 고정.
- **자동 생성**: 결과를 입력하면 이긴 팀이 다음 브라켓 자리로 올라가 **다음 라운드 대진이 자동 생성**. 준결승 패자 2팀으로 3·4위전 자동 생성.
- **결과 수정**: **가장 최근에 입력한 라운드만 되돌리기** 가능. 되돌리면 그 라운드는 미입력 상태로, 그 아래로 자동 생성됐던 다음 라운드는 초기화. (과거 임의 라운드 수정은 지원하지 않음 — 토너먼트 연쇄 붕괴 방지)

## 6. 화면 구성 (하단 탭 방식 / 모바일 우선)

하단 탭: **예측 · 대진표 · 순위** (+ 관리자는 별도 진입)

- **예측 탭**: 현재 열린 라운드의 경기 목록. 경기마다 두 팀 중 승팀 하나 선택 → "확정". 확정 후 마감 전이면 "확정 취소" 가능. 다른 사람은 확정 여부만 보임.
- **대진표 탭**: 전체 토너먼트 브라켓.
  - 결과 입력된 경기는 실제 진출 팀 표시, 미입력은 빈칸/미정.
  - **내 예측을 브라켓 위에 겹쳐 표시.** 맞으면 초록, 틀리면 빨강.
  - **마감 전 라운드**: 내 예측만. **마감된 라운드**: 4명 예측 모두 공개.
- **순위 탭**:
  - 상단: 총점·등수 리더보드(1등 강조).
  - 각 행 펼치기: 라운드별 획득 점수 분해(32강 ○점 / 16강 ○점 / …).
  - 하단: 선물 정산표.
- **관리자**: 별도 PIN 진입 → 셋업/대진/결과/마감/PIN 관리.

### 디자인 토큰 (기존 규칙 페이지에서 계승)

- 배경: 다크 그린 베이스(`#0c241d` 계열) + 잔디 줄무늬 + 은은한 축구장 라인(흰색 5~8% 투명도).
- 포인트: 잔디 그린 `#46e08a`, 트로피 골드 `#f4c64e`.
- 폰트: 디스플레이 **Do Hyeon**(제목·강 이름·점수), 본문 **Noto Sans KR**, 숫자/라벨 보조 **Oswald**.
- 톤: 스포티하지만 과하지 않게. 라인·줄무늬는 가독성 해치지 않을 만큼 은은하게.

## 7. 작업 단계 (이 순서로 진행)

### 1단계 — 프로젝트 뼈대 + DB 스키마 + 배포 README  ← **여기부터 시작**
- Next.js(App Router) 프로젝트 생성, Supabase 클라이언트 설정, 기본 레이아웃/탭 셸, 디자인 토큰(테마) 적용.
- Supabase 테이블 스키마 SQL 작성(아래 §8 데이터 모델 참고). RLS/관리자 검증 방침 명시.
- `README.md`에 **사용자가 직접 따라 하는 배포 순서**를 구체적으로: Supabase 프로젝트 생성 → SQL 실행 → 환경변수(.env) 설정 → 로컬 `npm run dev` 확인 → GitHub 푸시 → Vercel에서 레포 연결·환경변수 입력 → 배포 → URL 확인.
- 로컬에서 빌드/실행이 되는지 확인.

### 2단계 — 인증 + 관리자 셋업
- 이름+PIN 로그인, 관리자 PIN 진입.
- 관리자 셋업: 32강 16경기 브라켓 배치 UI, 참가자 4명 PIN 지정/재설정.

### 3단계 — 예측 + 마감 로직
- 라운드별 승팀 선택·확정·확정취소.
- 전원 확정 자동 마감 + 관리자 강제 마감.
- 마감 전 가림 규칙(확정 여부만 노출).

### 4단계 — 대진표 + 순위·채점·선물 정산
- 브라켓 뷰(내 예측 겹쳐보기, 마감 라운드 전원 공개, 맞음/틀림 색).
- 결과 입력 → 다음 라운드 자동 생성, 최근 라운드 되돌리기.
- 채점·동점·등수·선물 정산 자동 계산, 순위 화면.

## 8. 데이터 모델 (초안 — 1단계에서 확정)

> 아래는 출발점. Claude Code가 1단계에서 Supabase 스키마로 구체화하고 사용자와 확정한다.

- **participants**: 참가자 4명. `name`(윤/준/경/빈), `pin_hash`(개인 PIN 해시).
- **settings**: 단일 행. `admin_pin_hash`(관리자 PIN 해시), `current_open_round`(예측 오픈 라운드), `setup_done` 등.
- **matches**: 경기. `round`(R32/R16/R8/SF/FINAL/THIRD), `bracket_slot`(브라켓 위치), `team_a`, `team_b`, `winner`(결과·null이면 미입력), `is_locked`(예측 마감 여부).
- **predictions**: 예측. `participant`, `match_id`, `picked_team`, `confirmed`(확정 여부), `updated_at`. (참가자별 행 분리 → 동시 수정 충돌 방지)

채점/순위/선물은 저장값(예측·결과)으로부터 **계산해서** 보여준다(별도 점수 테이블 없이 파생). 필요 시 뷰/함수로.

### 라운드 키 정의
`R32`(32강·16경기, 1점) · `R16`(16강·8경기, 2점) · `R8`(8강·4경기, 4점) · `SF`(4강/준결승·2경기, 6점) · `FINAL`(결승·1경기, 10점) · `THIRD`(3·4위전·1경기, 4점)
